# Stage S3 — the hierarchy reduction layer

S3 is the hierarchy reduction layer. It consumes the S0 STRIDE table (the
profile — every locus evaluated at all seven hierarchy scales) and reduces it
*along the scale axis* into reusable, deterministic per-locus and per-chain
hierarchy products that later stages and figures consume.

> **What S3 is not.** S3 makes **no calibrated pass/fail resolution claim** — the
> gate ships uncalibrated (a provisional ρ\* = 0.5). Its products are
> **ρ\*-independent** descriptions of the profile: the ρ-vs-scale curve, the
> domain−residue gap, the monotonicity audit, and the chain contrast do not
> sweep a ρ\* band (that is S2/S5). S3 never recomputes ρ from its variance
> components, performs **no cross-serotype tests** (that is S5), does no ranking
> or machine learning, and produces no figures.

S3 never re-reads the raw STRIDE outputs, `profile.csv` / `mechanism.json`, or MD
trajectories. Its only input is the S0 STRIDE table.

---

## Input

| Input | Produced by | Columns S3 reads |
|---|---|---|
| `stride_table.parquet` | S0 | `serotype, canon_label, scale_level, scale_index, region_id, rho, gated, beta, coherence, h_chain, h_domain, is_gated_scale, mech_direction` |

The STRIDE table carries ρ at **all seven scales** for every locus (7 rows per
locus), so reducing along the scale axis is a filter over existing facts — never
a recomputation. ρ is read from the `rho` column and never reconstructed from its
variance components.

S3 is a **sibling reduction** to S2: both read the S0 profile independently. S2
reduces *within a serotype across a ρ\* band*; S3 reduces *along the scale axis*.
S3 does not consume S2's outputs.

---

## Core concepts

### The scale axis and upward closure

Every locus is profiled at seven scales, finest → coarsest: `residue (0),
secondary_structure (1), motif (2), domain (3), chain (4), protein (5), complex
(6)`. Reproducibility ρ is expected to be **non-decreasing** as the region
coarsens (the I2 upward-closure property): aggregating a locus into a larger
region should not lose reproducible signal. S3 audits this per locus and surfaces
the exceptions.

### Two tiers: licensed vs exploratory

At K = 3 replicates, only **domain-scale (and coarser)** claims are *licensed*;
**residue-scale** claims are *exploratory* and outside the operating range. Every
scale-curve row carries an explicit `tier` label, and the chain contrast reports
reproducibility at both the exploratory (residue) and licensed (chain) tiers.

### Distributed effects

A locus whose ρ is low at residue scale but high once aggregated to domain scale
is a **distributed effect** — reproducible only when aggregated. S3 flags it with
`is_distributed` when `rho_residue < ρ* ≤ rho_domain` under the provisional gate,
and always emits the raw Δρ so the flag can be re-derived at another threshold.

## Outputs

Five artifacts are written to the S3 output directory.

### 1. `scale_curve.parquet`

The ρ-vs-scale trajectory of every locus (figure F7).

- **Schema:** `serotype, canon_label, chain, domain, scale_index, scale_level, rho, rho_prev, rho_step_gain, rho_cumulative_gain, is_gated_scale, tier`
- **Primary key:** `(serotype, canon_label, scale_index)`
- **Producer:** S3 (`build_scale_curve`), one row per (locus, scale).
- **Downstream consumers:** the F7 ρ-vs-scale trajectories, and any analysis of
  how fast ρ climbs as a region coarsens (how spread out the reproducible signal
  is).
- **Invariants:** exactly one row per scale (0..6) per locus; `rho_prev` /
  `rho_step_gain` are null at residue scale; the profile's own gated scale is
  flagged on exactly one row per locus.

### 2. `resolution_gap.parquet`

The domain-vs-residue reproducibility gap, Δρ = ρ(domain) − ρ(residue).

- **Schema:** `serotype, canon_label, chain, domain, rho_residue, rho_domain, delta_rho_domain_residue, rho_min, rho_max, gated_scale_level, gated_scale_index, rho_at_gated, is_distributed, domain_tier`
- **Primary key:** `(serotype, canon_label)`
- **Producer:** S3 (`build_resolution_gap`).
- **Downstream consumers:** identification of distributed effects (reproducible
  only when aggregated) vs locally-resolved ones.
- **Invariants:** unique key; `delta_rho_domain_residue == rho_domain −
  rho_residue`; `domain_tier` is `licensed`.

### 3. `monotonicity_audit.parquet`

The upward-closure (I2) audit per locus.

- **Schema:** `serotype, canon_label, chain, domain, n_scales, is_monotone, n_violations, max_decrease, first_violation_scale_index, rho_residue, rho_complex`
- **Primary key:** `(serotype, canon_label)`
- **Producer:** S3 (`build_monotonicity_audit`).
- **Downstream consumers:** the I2 diagnostic — which loci violate upward closure
  (ρ drops when coarsening), how many steps, and by how much.
- **Invariants:** unique key; `n_scales == 7`; `is_monotone` iff `n_violations ==
  0`; a monotone row has `max_decrease == 0` and `first_violation_scale_index ==
  −1`; a non-monotone row has `max_decrease > 0`.

### 4. `chain_contrast.parquet`

The chain-level contrast (e.g. NS2B cofactor vs NS3 protease).

- **Schema:** `serotype, chain, n_loci, rho_residue_mean, rho_residue_median, rho_residue_min, rho_residue_max, rho_chain_mean, rho_chain_median, beta_residue_mean, n_mechanisms, n_increase, n_decrease, n_mixed, n_signed, tier`
- **Primary key:** `(serotype, chain)`
- **Producer:** S3 (`build_chain_contrast`), aggregating over the chain's member
  loci with **no region-constant assumption** (mean/median are well-defined even
  when coarse-scale ρ varies across a chain's loci).
- **Downstream consumers:** NS2B-vs-NS3 aggregate reproducibility and direction
  comparisons.
- **Invariants:** unique key; `n_increase + n_decrease + n_mixed == n_mechanisms`;
  `n_increase + n_decrease == n_signed`; `tier` is `licensed` (chain is coarser
  than domain).

### 5. `hierarchy_summary.json`

Machine-readable run summary: serotypes, table sizes, compact hierarchy facts
(number of distributed effects, monotone vs non-monotone locus counts), every
validation outcome, and the **provenance header** the design requires — the input
file SHA-256 digest, the provisional ρ\*, and `calibrated: false`. Consumed by CI
smoke checks and humans auditing a run.

---

## Validation (structural only)

Before the tables are trusted, S3 verifies — with no statistical assertions:

- **every output table's key is unique** (all four tables);
- **the scale curve is complete** — every locus carries exactly one row per scale
  (0..6), and the residue/domain ρ echoed by the gap table match the curve;
- **the resolution gap is arithmetically consistent** — Δρ equals `rho_domain −
  rho_residue` on every row;
- **the monotonicity audit is self-consistent** — the `is_monotone` flag matches
  the recomputed violation count, and the `max_decrease` / violation-index fields
  agree with it;
- **the chain contrast partitions each chain's mechanisms** — the direction
  counts sum to `n_mechanisms`, and increase + decrease equals `n_signed`.

Any failure raises a `ConsistencyError` (a subclass of `S3Error`); the run does
not produce partial outputs on failure.

## Usage

```bash
# after S0 has written the STRIDE table to outputs/
stride-s3 --input-dir outputs --output-dir outputs_s3
# or point at the table explicitly:
stride-s3 --stride-table outputs/stride_table.parquet --output-dir outputs_s3
```

Python API:

```python
from stride_s3 import run_s3, build_s3

# build + write artifacts
tables, report = run_s3("outputs/stride_table.parquet", "outputs_s3")

# build in memory only (no writes)
tables, report = build_s3("outputs/stride_table.parquet")
assert report.all_passed
tables.scale_curve          # the four tables are attributes of S3Tables
tables.resolution_gap
tables.monotonicity_audit
tables.chain_contrast
```

Reusing individual builders in a later stage:

```python
from stride_s3.build import (
    build_scale_curve,
    build_resolution_gap,
    build_monotonicity_audit,
    build_chain_contrast,
)
```

## Consumption map (quick reference)

| Table | Primary later consumers |
|---|---|
| `scale_curve` | F7 ρ-vs-scale trajectories; scale-of-resolution profiling |
| `resolution_gap` | distributed-effect identification (Δρ domain − residue) |
| `monotonicity_audit` | the I2 upward-closure diagnostic |
| `chain_contrast` | NS2B-vs-NS3 aggregate reproducibility and direction |

S3 hands later stages a trustworthy, purely structural reduction of the profile
along the scale axis. No calibrated verdicts, no cross-serotype tests, and no
inference happen here.
