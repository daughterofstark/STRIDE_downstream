# Stage S2 — the per-serotype reduction layer

S2 is the per-serotype reduction layer. It consumes the S0 STRIDE table and the
two S1B annotation tables, and derives reusable, deterministic per-serotype
reproducibility summaries that later stages consume.

> **What S2 is not.** S2 performs **no cross-serotype tests** — the serotype is
> the unit of biological replication, and any comparison *between* serotypes is
> deferred to S5. S2 makes **no calibrated pass/fail resolution claim**: the gate
> ships uncalibrated (a provisional ρ\* = 0.5), so every resolution statement is
> reported as a function of ρ\* over a **band**, never at a single threshold. S2
> never recomputes ρ from its variance components, does no machine learning or
> ranking, and produces no figures.

S2 never re-reads the raw STRIDE outputs, `profile.csv` / `mechanism.json`, or MD
trajectories. Its only inputs are the S0 STRIDE table and the S1B annotations.

---

## Inputs

| Input | Produced by | Columns S2 reads |
|---|---|---|
| `stride_table.parquet` | S0 | `serotype, canon_label, scale_level, scale_index, region_id, rho, gated, beta, beta_se, tau2, sigma2_bar, coherence, h_chain, h_domain, is_gated_scale, mech_direction, mech_beta_signed, mech_beta_ci_lower, mech_beta_ci_upper, mech_reproducible_magnitude_energy` |
| `residue_annotation.parquet` | S1B | `serotype, canon_label, chain, domain, domain_status, conservation_class` |
| `domain_annotation.parquet` | S1B | `serotype, chain, domain, domain_status, n_residues` |

The STRIDE table already carries ρ (and, on gated rows, the signed mechanism
payload) at **all seven hierarchy scales**, so re-gating at any ρ\* is a filter
over existing facts — never a recomputation.

---

## Core concepts

### The ρ\* band, not a single threshold

The gate is **uncalibrated**. Rather than commit to one ρ\* and emit a pass/fail
verdict, S2 sweeps a band of ρ\* values (the frozen default is `0.50 → 0.90` in
steps of `0.05`) and reports every resolution/screen quantity as a function of
ρ\*. The provisional ρ\* = 0.5 that ships in the mechanism files is flagged on
each row (`is_provisional_rho_star`) but never privileged.

### Re-gating

A locus **gates at the finest scale** (smallest scale index) at which `ρ ≥ ρ*`.
Raising ρ\* can only push a locus coarser (or leave it unresolved), never finer —
a monotonicity S2 validates. Scale indices run `residue=0, secondary_structure=1,
motif=2, domain=3, chain=4, protein=5, complex=6`.

### Two tiers: licensed vs exploratory

At K = 3 replicates, only **domain-scale (and coarser)** claims are *licensed*;
**residue-scale** claims are *exploratory* and outside the operating range. Every
output row carries an explicit `tier` label so a residue-scale number can never
be mistaken for a licensed claim.

### The signed / significant screen

A gated mechanism yields a signed, significant, reproducible claim when its
direction is not `mixed` **and** its β confidence interval strictly excludes 0
**and** its ρ meets ρ\*. Mixed mechanisms carry a real unsigned magnitude but no
signed direction, so they never pass the screen.

## Outputs

Six artifacts are written to the S2 output directory.

### 1. `resolution_census.parquet`

The achieved-resolution census, swept over the band.

- **Schema:** `serotype, rho_star, is_provisional_rho_star, gated_scale_level, gated_scale_index, tier, n_loci`
- **Primary key:** `(serotype, rho_star, gated_scale_level)`
- **Producer:** S2 (`build_resolution_census`), by re-gating every locus at each
  ρ\* in the band and counting loci per gated scale.
- **Downstream consumers:** any stage or report that asks "at ρ\* = x, how many
  loci in this serotype resolve at domain scale vs residue scale vs not at all".
- **Invariants:** at every `(serotype, ρ*)` the per-scale counts sum to the
  serotype's locus count (the census is *total*); the residue-gated count is
  non-increasing as ρ\* rises.

### 2. `residue_landscape.parquet` (Tier B — exploratory)

The residue-scale reproducibility landscape.

- **Schema:** `serotype, canon_label, chain, domain, domain_status, conservation_class, rho_residue, beta_residue, coherence_residue, tau2_residue, sigma2_bar_residue, gated_scale_level_provisional, gated_scale_index_provisional, gates_at_residue_provisional, tier`
- **Primary key:** `(serotype, canon_label)`
- **Producer:** S2 (`build_residue_landscape`), from the residue-scale profile
  rows joined to the S1B residue annotation.
- **Downstream consumers:** exploratory residue-level views of ρ and the finest
  scale each locus resolves at under the provisional gate.
- **Invariants:** one row per canonical residue; every row is `exploratory`.

### 3. `domain_reproducibility.parquet` (Tier A — licensed)

Domain-scale reproducibility, the licensed claim level.

- **Schema:** `serotype, chain, domain, domain_status, region_id, n_residues, rho_domain, beta_domain, coherence_domain, tau2_domain, sigma2_bar_domain, is_coherent, tier`
- **Primary key:** `(serotype, chain, domain)`
- **Producer:** S2 (`build_domain_reproducibility`), from the domain-scale profile
  rows joined to the S1B domain annotation.
- **Downstream consumers:** licensed, domain-granularity reproducibility claims.
- **Invariants:** unique key; a domain region's domain-scale ρ / β / coherence are
  region-level constants across the member loci (verified — a non-constant region
  raises); `is_coherent` iff coherence ≥ 0.6; every row is `licensed`.

### 4. `signed_screen.parquet`

The signed / significant screen over the gated mechanisms, swept over the band.

- **Schema:** `serotype, canon_label, rho_star, is_provisional_rho_star, gated_scale_level, tier, rho, direction, is_signed, beta_signed, beta_ci_lower, beta_ci_upper, ci_excludes_zero, meets_rho_star, passes_screen`
- **Primary key:** `(serotype, canon_label, rho_star)`
- **Producer:** S2 (`build_signed_screen`), by evaluating the screen at every ρ\*.
- **Downstream consumers:** any stage reporting which mechanisms are signed and
  significant as a function of ρ\*.
- **Invariants:** unique key; `passes_screen` iff `is_signed ∧ ci_excludes_zero ∧
  meets_rho_star`.

### 5. `serotype_summary.parquet`

The per-serotype scorecard, swept over the band.

- **Schema:** `serotype, rho_star, is_provisional_rho_star, n_loci, n_gated_residue, n_gated_domain_or_coarser, n_unresolved, frac_gated_residue, n_mechanisms, n_signed, n_mixed, n_signed_significant, frac_mixed, rho_residue_median, rho_residue_q1, rho_residue_q3, rho_residue_min, rho_residue_max`
- **Primary key:** `(serotype, rho_star)`
- **Producer:** S2 (`build_serotype_summary`), rolling up the census, residue
  landscape and signed screen so the scorecard cannot disagree with them.
- **Downstream consumers:** dataset-level per-serotype reproducibility summaries.
- **Invariants:** unique key; `n_signed + n_mixed == n_mechanisms`; the gated-count
  partition never exceeds `n_loci`; `n_loci` agrees with the census total.

### 6. `reduction_summary.json`

Machine-readable run summary: serotypes, the ρ\* band, table sizes, compact
per-serotype facts at the provisional ρ\*, every validation outcome, and the
**provenance header** the design requires — input file SHA-256 digests, the ρ\*
band, and `calibrated: false`. Consumed by CI smoke checks and humans auditing a
run.

---

## Validation (structural only)

Before the tables are trusted, S2 verifies — with no statistical assertions:

- **every output table's key is unique** (all five tables);
- **the resolution census is total** — its per-scale counts sum to the serotype's
  locus count at every ρ\*;
- **re-gating is monotone in ρ\*** — the residue-gated count never increases as
  ρ\* rises;
- **the per-serotype scorecard partitions the mechanisms** (`signed + mixed =
  n_mechanisms`) and its census rollups agree with the census table;
- **tier labels are correct** — residue-scale rows are `exploratory`, domain-scale
  rows are `licensed`.

Any failure raises a `ConsistencyError` (a subclass of `S2Error`); the run does
not produce partial outputs on failure.

## Usage

```bash
# after S0 has written the STRIDE table to outputs/ and S1B its annotations to outputs_s1b/
stride-s2 --stride-input-dir outputs --annotation-input-dir outputs_s1b --output-dir outputs_s2
# override the band, or point at individual tables:
stride-s2 --stride-table outputs/stride_table.parquet \
          --residue-annotation outputs_s1b/residue_annotation.parquet \
          --domain-annotation outputs_s1b/domain_annotation.parquet \
          --rho-star 0.5 0.6 0.7 0.8 0.9 \
          --output-dir outputs_s2
```

Python API:

```python
from stride_s2 import run_s2, build_s2

# build + write artifacts
tables, report = run_s2(
    "outputs/stride_table.parquet",
    "outputs_s1b/residue_annotation.parquet",
    "outputs_s1b/domain_annotation.parquet",
    "outputs_s2",
)

# build in memory only (no writes); optionally supply a custom ρ* band
tables, report = build_s2(st_path, ra_path, da_path, rho_star_band=[0.5, 0.7, 0.9])
assert report.all_passed
tables.resolution_census        # the five tables are attributes of S2Tables
tables.residue_landscape
tables.domain_reproducibility
tables.signed_screen
tables.serotype_summary
```

Reusing individual builders in a later stage:

```python
from stride_s2.build import (
    build_resolution_census,
    build_domain_reproducibility,
    build_signed_screen,
)
```

## Consumption map (quick reference)

| Table | Tier | Primary later consumers |
|---|---|---|
| `resolution_census` | mixed | achieved-resolution counts as a function of ρ\* |
| `residue_landscape` | exploratory | residue-scale ρ views (outside the operating range) |
| `domain_reproducibility` | licensed | domain-scale reproducibility claims at K = 3 |
| `signed_screen` | mixed | signed/significant mechanism screen over the band |
| `serotype_summary` | mixed | per-serotype reproducibility scorecards |

S2 hands later stages a trustworthy, purely per-serotype reduction layer reported
over a ρ\* band. No cross-serotype tests, no calibrated verdicts, and no inference
happen here.
