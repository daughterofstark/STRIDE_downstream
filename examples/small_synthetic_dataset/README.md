# small_synthetic_dataset

A tiny, fully synthetic dataset in the exact layout the framework discovers. It
is **committed on purpose** (unlike real data) so the repository runs out of the
box:

```bash
stride-s0 --data-root examples/small_synthetic_dataset --output-dir outputs
```

## What it contains

- **4 serotypes** (`DENV1`–`DENV4`) × **3 replicates** (`1st_run`, `2nd_run`,
  `3rd_run`) of Level-1 `*_correlations_v5.csv`, each with **2 residues**.
- Matching Level-2 `*_profile.csv` + `*_mechanism.json` under `summaries/`.

Running S0 on it yields a **24-row replicate table** (4 × 3 × 2) and a
**56-row STRIDE table** (4 × 2 loci × 7 scales), with all checks passing.

## Layout

```
small_synthetic_dataset/
├── 1st_run/DENV1/analysis_output/DENV1_correlations_v5.csv
├── 1st_run/DENV2/... (DENV3, DENV4)
├── 2nd_run/... 3rd_run/...
└── summaries/
    ├── DENV1_profile.csv
    ├── DENV1_mechanism.json
    └── ... (DENV2–4)
```

## Notes

- The numbers are arbitrary but internally consistent (they satisfy every S0
  invariant: one gated row per locus, gated ρ matches the mechanism, null-iff-
  mixed betas, etc.). They carry **no biological meaning**.
- The data is generated by `stride_analysis._synthetic.write_dataset`. To
  regenerate it:

  ```python
  from stride_analysis._synthetic import write_dataset
  from pathlib import Path
  write_dataset(Path("examples/small_synthetic_dataset"),
                ["DENV1", "DENV2", "DENV3", "DENV4"],
                ["1st_run", "2nd_run", "3rd_run"])
  ```
