# Stage S7 ‚Äî the reporting layer (figures & tables)

S7 is the terminal, **reporting-only** stage. It consumes the already-reduced
outputs of **S2‚ÄìS6** and assembles the design's publication figures (**F1‚ÄìF8**,
design ¬ß3.7) and manuscript tables (**T1‚ÄìT5**, design ¬ß3.8), deterministically and
provenance-stamped (design stage table, row S7: *"all above ‚Üí F1‚ÄìF8, T1‚ÄìT5.
Deterministic, provenance-stamped."*).

> **What S7 is not.** S7 performs **no new statistics and no biological
> inference**. Every value it emits is read from a prior stage's table and merely
> selected, joined, ordered, and formatted. It never reads the raw STRIDE
> `profile.csv` / `mechanism.json`, never re-reads the S0/S1 canonical layer, and
> never recomputes a quantity an earlier stage already produced. It makes **no
> calibrated pass/fail claim** ‚Äî figures and tables inherit the uncalibrated gate
> (`calibrated = false`, provisional œÅ\* = 0.5) and the two-tier
> (`licensed` / `exploratory`) labels of their source rows.

S7 never modifies an earlier stage.

## Tables (T1‚ÄìT5) ‚Äî all fully supported

| Tbl | Table | Source table(s) (stage) |
| --- | --- | --- |
| T1 | Per-serotype summary | `serotype_summary` (S2) |

T2 is a left join on `(serotype, chain, domain)`; T4 **ranks** shared positions by

reproducible fraction and median œÅ) and keeps the top `T4_TOP_N = 25` ‚Äî a sort over
existing columns, not a new metric.

## The replicate layer (S6) feeds no design figure/table

None of F1‚ÄìF8 or T1‚ÄìT5 draws on the per-replicate axis. Rather than fabricate a
figure the design does not specify, S7 **documents** the replicate layer: S6's
`replicate_regime` and `replicate_blocked_analyses` are surfaced under the summary's
`limitations`, and both S6 tables are digested into the provenance header. The
blocked per-run analyses (per-run rank concordance / effect spread when the per-run
correlation CSVs are absent; leave-one-replicate-out stability) are thus recorded,
not approximated.

---

## Figures are deterministic, dependency-free SVG

Figures are emitted as hand-rendered **SVG** (vector, publication-quality) built
from primitives in `plotting/svg.py`, alongside their **prepared-data** `.csv` +
`.parquet` (the plotting-ready table each figure was rendered from). The builders in
`build/` prepare the data; the renderers in `plotting/` only lay out marks from
already-prepared tables ‚Äî plotting never touches a source table and computes
nothing.

**Why SVG only (no PNG/PDF).** PNG/PDF rasterisation would require a heavyweight,
version-sensitive rendering dependency (e.g. matplotlib) whose byte output is **not
reproducible** across environments ‚Äî directly conflicting with this framework's
determinism guarantee and its no-unnecessary-dependencies convention (SciPy was
likewise avoided in S4/S6). SVG is an explicitly permitted, dependency-free vector
format that is trivially byte-reproducible and converts to PNG/PDF downstream if a
consumer needs raster. This is a deliberate **format** choice, **not** a
data-availability limitation.

Determinism is achieved by sorting rows, rounding floats to a fixed precision,
formatting all coordinates to two decimals, and embedding **no** wall-clock
timestamp. Running S7 twice on the same inputs yields **byte-identical** output. The
design's provenance "date" field is intentionally omitted for exactly this reason.

## Outputs

Per figure `FN`: `<slug>.svg` + `<slug>.csv` + `<slug>.parquet`.
Per table `TN`: `<slug>.csv` + `<slug>.parquet` + `<slug>.md` (GitHub-flavoured
Markdown). Plus:

- `artifact_manifest.parquet` ‚Äî one row per generated artifact (id, kind, title,
  slug, sources, files, row count, tier).
- `report_summary.json` ‚Äî the machine-readable run report: the provenance header
  (input SHA-256 digests, provisional œÅ\*, `calibrated = false`, the K = 3 note, the
  SVG/determinism notes), the generated-artifact records, the replicate-layer
  `limitations`, and the structural `checks`.

Filenames are deterministic slugs (`F1_reproducibility_landscape`,
`T3_catalytic_cross_serotype`, ‚Ä¶).

---

## Validation (structural only)

S7 validates structure, never scientific conclusions:

- **completeness** ‚Äî every design figure (F1‚ÄìF8) and table (T1‚ÄìT5) is present once;
- **columns** ‚Äî each prepared figure frame and each table carries exactly its
  declared schema;
- **filenames** ‚Äî every output filename is the deterministic `<slug><suffix>`, and
  each figure has `.svg` + `.csv` + `.parquet` while each table has `.csv` +
  `.parquet` + `.md`;
- **on-disk** ‚Äî every recorded artifact exists in the output directory;
- **provenance** ‚Äî the header carries `calibrated = false`, the provisional œÅ\*, the
  K note, and a per-input SHA-256 digest.
