# Stage S4 — the uncertainty layer

S4 is the uncertainty layer. It consumes the S0 STRIDE table (the profile —
every locus evaluated at all seven hierarchy scales, carrying the variance
components τ² and σ̄² and the gated mechanism payload) and derives reusable,
deterministic per-domain and per-residue **uncertainty products**: where a
region's non-reproducibility originates (replicate disagreement vs sampling), how
strongly the runs disagree, which signed mechanisms survive a confidence-interval
screen under false-discovery-rate control, and the precision-weighted average
effect of each domain.

> **What S4 is not.** S4 makes **no calibrated pass/fail resolution claim** — the
> gate ships uncalibrated (a provisional ρ\* = 0.5). Its products are
> **ρ\*-independent** descriptions of the profile and the emitted mechanisms: the
> variance budget, the τ² ranking, the CI significance screen, and the
> β_se-weighted summaries do not sweep a ρ\* band (that is S2/S5). S4 never
> recomputes ρ or the variance components, performs **no cross-serotype tests**
> (that is S5 — the serotype is the unit of replication and therefore the FDR
> family), does no machine learning, and produces no figures.

S4 never re-reads the raw STRIDE outputs, `profile.csv` / `mechanism.json`, or MD
trajectories. Its only input is the S0 STRIDE table.

---

## Input

| Input | Produced by | Columns S4 reads |
|---|---|---|
| `stride_table.parquet` | S0 | `serotype, canon_label, scale_level, scale_index, region_id, rho, gated, beta, beta_se, tau2, sigma2_bar, coherence, h_chain, h_domain, is_gated_scale, mech_direction, mech_beta_signed, mech_beta_ci_lower, mech_beta_ci_upper, mech_beta_se` |

The STRIDE table carries the variance components τ² and σ̄² at every scale for
every locus, and the signed-effect payload (`mech_beta_signed`, its SE, and its
CI) on each gated row. S4 reads these directly — τ², σ̄², β, β_se, and the CI are
**never recomputed**.

S4 is a **sibling reduction** to S2 and S3: all three read the S0 profile
independently. S2 reduces *within a serotype across a ρ\* band*; S3 reduces
*along the scale axis*; S4 reduces the profile's *uncertainty* — the variance
components and the mechanism confidence intervals. S4 does not consume S2's or
S3's outputs.

---

## Core concepts

### The variance components τ² and σ̄²

Each region's non-reproducibility splits into two read-from-profile pieces: **τ²**
is the between-replicate variance (how much the three runs disagree), and **σ̄²**
is the autocorrelation-corrected within-replicate variance (sampling noise). Their
sum is the region's *unreproduced* variance. S4 reports the two **fractions** of
that total and their **ratio**, never reconstructing ρ from them.

### The τ²/σ̄² ratio diagnostic

A region with a **high τ² fraction** is *replicate-dominated*: the runs genuinely
disagree (candidate metastability), and more sampling within a run would not help.
A region with a **low τ² fraction** is *sampling-dominated*: the disagreement is
undersampling, and longer runs would help. S4 labels each region
`replicate_dominated` (τ² fraction ≥ 0.6), `sampling_dominated` (≤ 0.4), or
`balanced`; a region with zero unreproduced variance is `undefined`.

### Two tiers: licensed vs exploratory

At K = 3 replicates, only **domain-scale (and coarser)** claims are *licensed*;
**residue-scale** claims are *exploratory* and outside the operating range. The
per-domain variance budget and the domain effect summary are Tier A (licensed);
the per-residue variance map is Tier B (exploratory). The significance screen
carries the tier of each mechanism's own gated scale.

### CI-based significance with FDR control

A signed mechanism is **significant (raw)** when its effect confidence interval
excludes 0 — the design's CI screen. Because a serotype has many positions,
reporting raw counts would inflate false positives, so S4 also computes a
two-sided Wald p-value (`z = β/β_se`, normal tail — no SciPy dependency) and
applies **Benjamini–Hochberg** FDR control **across the signed mechanisms of each
serotype**. `significant_fdr` is the BH-adjusted decision at level α = 0.05. The
serotype is the multiple-testing family because it is the unit of replication.

### β_se-weighted summaries

When averaging effects within a domain, precise estimates should count more, so
S4 weights each signed effect by 1/β_se² (inverse-variance weighting) and reports
the weighted mean, its standard error √(1/Σ w), and the plain unweighted mean
alongside it.

## Outputs

Five artifacts are written to the S4 output directory.

### 1. `variance_budget.parquet`

The per-domain variance-component budget (τ² vs σ̄²) — Tier A, licensed (table T5).

- **Schema:** `serotype, chain, domain, region_id, rho_domain, beta_domain, beta_se_domain, tau2, sigma2_bar, total_unreproduced, frac_tau2, frac_sigma2, tau2_sigma2_ratio, variance_regime, tier`
- **Primary key:** `(serotype, chain, domain)`
- **Producer:** S4 (`build_variance_budget`), reading each domain region's
  region-constant domain-scale row.
- **Downstream consumers:** the stacked τ²/σ̄² budget figure and the
  replicate-vs-sampling attribution per domain.
- **Invariants:** unique key; `total_unreproduced == tau2 + sigma2_bar`;
  `frac_tau2 + frac_sigma2 == 1` where the total is positive; `variance_regime`
  matches `frac_tau2`; `tier` is `licensed`.

### 2. `residue_variance.parquet`

The per-residue variance decomposition and replicate-disagreement ranking — Tier
B, exploratory.

- **Schema:** `serotype, canon_label, chain, domain, rho_residue, beta_residue, beta_se_residue, tau2, sigma2_bar, total_unreproduced, frac_tau2, frac_sigma2, tau2_sigma2_ratio, variance_regime, tau2_rank, tier`
- **Primary key:** `(serotype, canon_label)`
- **Producer:** S4 (`build_residue_variance`); `tau2_rank` orders positions
  within a serotype by τ² descending (rank 1 = the runs disagree most).
- **Downstream consumers:** the replicate-disagreement map — which residues to
  scrutinise for run-to-run instability.
- **Invariants:** unique key; fractions consistent as above; `tau2_rank` is a
  dense 1-based ranking within each serotype; `tier` is `exploratory`.

### 3. `significance_screen.parquet`

The CI-based significance screen over gated mechanisms, with BH-FDR control.

- **Schema:** `serotype, canon_label, chain, domain, gated_scale_level, tier, direction, is_signed, beta_signed, beta_se, beta_ci_lower, beta_ci_upper, ci_excludes_zero, z_score, p_value, p_value_bh, significant_raw, significant_fdr`
- **Primary key:** `(serotype, canon_label)`
- **Producer:** S4 (`build_significance_screen`), one row per gated mechanism;
  BH is applied within each serotype over its signed mechanisms.
- **Downstream consumers:** the signed-significant-reproducible shortlist and the
  per-serotype false-discovery-controlled counts.
- **Invariants:** unique key; `is_signed` iff `direction != mixed`; unsigned rows
  carry no CI exclusion, no BH p-value, and no FDR flag; `significant_raw ==
  ci_excludes_zero`; BH-adjusted p-values are monotone in raw-p order within a
  serotype.

### 4. `domain_effect_summary.parquet`

The β_se-weighted effect summary per domain — Tier A, licensed.

- **Schema:** `serotype, chain, domain, n_mechanisms, n_signed, n_mixed, n_ci_excludes_zero, frac_ci_excludes_zero, n_significant_fdr, beta_weighted_mean, beta_weighted_se, beta_unweighted_mean, tier`
- **Primary key:** `(serotype, chain, domain)`
- **Producer:** S4 (`build_domain_effect_summary`), aggregating the significance
  screen by the gated mechanism's domain.
- **Downstream consumers:** the per-domain fraction of signed mechanisms whose CI
  excludes 0, and the precision-weighted average domain effect.
- **Invariants:** unique key; `n_signed + n_mixed == n_mechanisms`;
  `n_ci_excludes_zero` and `n_significant_fdr` never exceed `n_signed`;
  `frac_ci_excludes_zero ∈ [0, 1]`; `tier` is `licensed`.

### 5. `uncertainty_summary.json`

Machine-readable run summary: serotypes, table sizes, compact uncertainty facts
(domain variance-regime counts, signed / CI-excluding / FDR-significant mechanism
counts), every validation outcome, and the **provenance header** the design
requires — the input file SHA-256 digest, the provisional ρ\*, the FDR level α,
and `calibrated: false`. Consumed by CI smoke checks and humans auditing a run.

## Validation (structural only)

Before the tables are trusted, S4 verifies — with no statistical assertions:

- **every output table's key is unique** (all four tables);
- **the variance fractions are arithmetically consistent** — where the
  unreproduced variance is positive, `frac_tau2 + frac_sigma2 == 1`, both lie in
  `[0, 1]`, and the `variance_regime` label matches the recomputed τ² fraction
  (a `nan` fraction is allowed only when the regime is `undefined`);
- **the significance screen is self-consistent** — `is_signed` matches the
  direction, unsigned rows carry no CI/BH/FDR decision, `significant_raw` equals
  the CI-exclusion flag, and the BH-adjusted p-values are monotone in raw-p order
  within each serotype;
- **the domain effect summary partitions each domain's mechanisms** — the signed
  and mixed counts sum to `n_mechanisms`, and the CI/FDR counts never exceed
  `n_signed`.

Any failure raises a `ConsistencyError` (a subclass of `S4Error`); the run does
not produce partial outputs on failure.

## Usage

```bash
# after S0 has written the STRIDE table to outputs/
stride-s4 --input-dir outputs --output-dir outputs_s4
# or point at the table explicitly:
stride-s4 --stride-table outputs/stride_table.parquet --output-dir outputs_s4
```

Python API:

```python
from stride_s4 import run_s4, build_s4

# build + write artifacts
tables, report = run_s4("outputs/stride_table.parquet", "outputs_s4")

# build in memory only (no writes)
tables, report = build_s4("outputs/stride_table.parquet")
assert report.all_passed
tables.variance_budget          # the four tables are attributes of S4Tables
tables.residue_variance
tables.significance_screen
tables.domain_effect_summary
```

Reusing individual builders in a later stage:

```python
from stride_s4.build import (
    build_variance_budget,
    build_residue_variance,
    build_significance_screen,
    build_domain_effect_summary,
)
```

## Consumption map (quick reference)

| Table | Primary later consumers |
|---|---|
| `variance_budget` | stacked τ²/σ̄² budget; replicate-vs-sampling attribution per domain |
| `residue_variance` | the replicate-disagreement map (τ² ranking) |
| `significance_screen` | signed-significant-reproducible shortlist; FDR-controlled counts |
| `domain_effect_summary` | per-domain CI-exclusion fraction; β_se-weighted domain effect |

S4 hands later stages a trustworthy, purely structural reduction of the profile's
uncertainty. No calibrated verdicts, no cross-serotype tests, and no inference
happen here.
