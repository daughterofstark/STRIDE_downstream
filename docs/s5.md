# Stage S5 — the cross-serotype layer (n = 4)

S5 is the cross-serotype layer. It consumes the S0 STRIDE table (the profile —
every locus at all seven hierarchy scales, carrying ρ, the gated mechanism
direction, and the variance components) together with the S1A
`conservation_table` (the shared-position index — which serotypes contain each
`canon_label`), and compares the **four** dengue serotypes at the level of shared
canonical positions and named regions: whether reproducibility is **conserved**
across serotypes, whether signed effects **agree in direction**, how the domains
compare in a **ρ(domain × serotype)** matrix, and a per-serotype **scorecard**.

> **What S5 is not.** S5 treats **serotype as the unit of biological replication
> (n = 4)**. It aggregates each serotype to one value per position/region **first**,
> then compares across the four serotypes — it never treats residues or frames as
> independent samples, and it reports **descriptive** statistics and effect sizes
> (counts, fractions, majority tallies) rather than p-values across residues. It
> makes **no calibrated pass/fail resolution claim** — the gate ships uncalibrated
> (a provisional ρ\* = 0.5), so every reproducibility statement is descriptive and
> labelled provisional. S5 never recomputes ρ or the variance components, does no
> machine learning, and produces no figures.

S5 reads only the S0 STRIDE table and the S1A conservation table. It never
consumes the S2/S3/S4 reduction outputs, never re-reads the raw STRIDE
`profile.csv` / `mechanism.json` or MD trajectories, and never modifies an earlier
stage.

---

## Input

| Input | Produced by | Columns S5 reads |
|---|---|---|
| `stride_table.parquet` | S0 | `serotype, canon_label, scale_level, scale_index, region_id, rho, gated, beta, beta_se, tau2, sigma2_bar, h_chain, h_domain, is_gated_scale, mech_direction` |
| `conservation_table.parquet` | S1A | `canon_label, n_serotypes, serotypes_present, in_all_serotypes, chain, domain` |

The STRIDE table supplies each locus's residue-scale ρ, its gated mechanism
direction, and the region-constant domain-scale ρ/β/variance components. The
conservation table supplies the authoritative cross-serotype presence map (the
199-shared-position index in the real data). ρ and the variance components are
**read**, never recomputed.

---

## Core concepts

### Reproducible at the provisional gate

A position is **reproducible** in a serotype when its residue-scale ρ clears the
provisional threshold ρ\* = 0.5. Because the gate marks the finest scale where
ρ ≥ ρ\*, this is exactly equivalent to the locus being **gated at the residue
scale**. The gate is uncalibrated, so this is a *descriptive* determination,
stamped `is_provisional_rho_star = true`; ρ\* can be overridden (`--rho-star`) to
sweep the band.

### Signed at the provisional gate

A position is **signed** in a serotype when it is reproducible *and* the direction
of its (residue-scale) gated mechanism is `increase` or `decrease`. A
reproducible-but-`mixed` position carries a real magnitude but no signed claim, so
it is *not* signed.

### Aggregate first (the n = 4 guardrail)

Every cross-serotype product is built from the per-`(serotype, position)`
aggregation (`position_frame`) or the per-`(serotype, domain)` region collapse
(`domain_regions`). Serotype counts are then tallied across those aggregates. No
step treats the ~200 residues of a serotype as independent samples.

### Two tiers: licensed vs exploratory

At K = 3 replicates only **domain-scale (and coarser)** claims are *licensed*;
**residue-scale** claims are *exploratory*. The domain × serotype matrix is Tier A
(licensed); the position-conservation, direction-concordance, and scorecard tables
are residue-scale, hence Tier B (exploratory).

### Conserved catalytic machinery

The **Catalytic Triad** residues (`NS3:51`, `NS3:75`, `NS3:135`) are flagged in the
position tables, and the catalytic-machinery **domains** (Catalytic Triad, Oxyanion
Loop) are flagged in the matrix, so their cross-serotype behaviour — expected to be
the most conserved — can be read off directly.

## Outputs

Five artifacts are written to the S5 output directory.

### 1. `position_conservation.parquet`

Conservation of reproducibility across the shared positions — Tier B, exploratory.

- **Schema:** `canon_label, chain, domain, n_serotypes_total, n_serotypes_present, serotypes_present, in_all_serotypes, n_serotypes_reproducible, n_serotypes_signed_reproducible, frac_reproducible, conservation_class, is_serotype_divergent, is_catalytic_triad, rho_residue_min, rho_residue_median, rho_residue_max, rho_star, is_provisional_rho_star, tier`
- **Primary key:** `(canon_label)`
- **Producer:** S5 (`build_position_conservation`), joining the per-serotype
  reproducibility of each shared position onto the S1A presence map.
- **Downstream consumers:** the cross-serotype conservation map (F5) and the
  serotype-divergent-position shortlist.
- **Invariants:** unique key; `0 ≤ signed ≤ reproducible ≤ present`;
  `frac_reproducible = reproducible / present`; `conservation_class` matches the
  counts (`reproducible_all` / `majority` / `some` / `none`);
  `is_serotype_divergent` iff signed-and-reproducible in some but not all
  serotypes; `is_catalytic_triad` matches the Triad label set; `tier` is
  `exploratory`.

### 2. `direction_concordance.parquet`

Direction agreement for shared signed positions — Tier B, exploratory.

- **Schema:** `canon_label, chain, domain, n_serotypes_signed, n_increase, n_decrease, majority_direction, frac_majority, concordance_class, is_catalytic_triad, rho_star, is_provisional_rho_star, tier`
- **Primary key:** `(canon_label)`
- **Producer:** S5 (`build_direction_concordance`); only positions signed in ≥ 2
  serotypes appear (so concordance is well-posed).
- **Downstream consumers:** the direction-concordance view over shared signed
  positions.
- **Invariants:** unique key; `n_increase + n_decrease == n_serotypes_signed ≥ 2`;
  `concordance_class` is `agree` (one direction only), `majority` (both, one
  strictly larger), or `conflict` (a tie); `majority_direction` and `frac_majority`
  match the counts; `tier` is `exploratory`.

### 3. `domain_serotype_matrix.parquet`

The tidy-long ρ(domain × serotype) matrix — Tier A, licensed (figure F3).

- **Schema:** `serotype, chain, domain, region_id, rho_domain, beta_domain, beta_se_domain, tau2_domain, sigma2_bar_domain, is_catalytic_domain, tier`
- **Primary key:** `(serotype, chain, domain)`
- **Producer:** S5 (`build_domain_serotype_matrix`), reading each domain region's
  region-constant domain-scale row; a later stage pivots it to the heatmap.
- **Downstream consumers:** the domain × serotype ρ heatmap; the conserved
  catalytic-machinery check.
- **Invariants:** unique key; `rho_domain ∈ [0, 1]`; `is_catalytic_domain` matches
  the catalytic-domain set; `tier` is `licensed`.

### 4. `cross_serotype_scorecard.parquet`

The per-serotype cross-serotype scorecard — Tier B, exploratory (table T1).

- **Schema:** `serotype, n_loci, n_reproducible_residue, frac_reproducible_residue, n_mechanisms, n_signed, n_mixed, frac_signed, frac_mixed, rho_residue_median, rho_residue_min, rho_residue_max, n_shared_positions, n_shared_reproducible, rho_star, is_provisional_rho_star, tier`
- **Primary key:** `(serotype)`
- **Producer:** S5 (`build_cross_serotype_scorecard`); mechanisms are counted one
  per gated region.
- **Downstream consumers:** the per-serotype summary row and the shared-position
  reproducibility comparison.
- **Invariants:** unique key; `n_signed + n_mixed == n_mechanisms`;
  `n_reproducible_residue ≤ n_loci`; `n_shared_reproducible ≤ n_shared_positions`;
  every fraction in `[0, 1]`; `tier` is `exploratory`.

### 5. `conservation_summary.json`

Machine-readable run summary: serotypes, table sizes, compact cross-serotype facts
(conservation-class counts, divergent-position count, Catalytic-Triad position
count, concordance-class counts, catalytic-domain cell count), every validation
outcome, and the **provenance header** the design requires — the two input file
SHA-256 digests, the provisional ρ\*, `calibrated: false`, and the n = 4
replication note.

## Validation (structural only)

Before the tables are trusted, S5 verifies — with no statistical assertions:

- **every output table's key is unique** (all four tables);
- **position conservation is arithmetically consistent** — the counts are ordered
  `0 ≤ signed ≤ reproducible ≤ present`, the reproducible fraction matches the
  counts, the `conservation_class` and `is_serotype_divergent` labels match the
  recomputed counts, and the Catalytic-Triad flag matches the label set;
- **direction concordance partitions each position's signed serotypes** —
  `n_increase + n_decrease == n_serotypes_signed` (at least the required minimum),
  and the `concordance_class` / `majority_direction` / `frac_majority` match;
- **the domain × serotype matrix** carries ρ in `[0, 1]`, is Tier A, and its
  catalytic-domain flag matches the label set;
- **the scorecard partitions each serotype's mechanisms** (`n_signed + n_mixed ==
  n_mechanisms`) with the residue / shared counts bounded and every fraction in
  `[0, 1]`.

Any failure raises a `ConsistencyError` (a subclass of `S5Error`); the run does not
produce partial outputs on failure.

## Usage

```bash
# after S0 wrote the STRIDE table to outputs/ and S1A wrote its tables to outputs_s1a/
stride-s5 --input-dir outputs --conservation-input-dir outputs_s1a --output-dir outputs_s5
# or point at the two tables explicitly:
stride-s5 --stride-table outputs/stride_table.parquet \
          --conservation-table outputs_s1a/conservation_table.parquet \
          --output-dir outputs_s5
```

Python API:

```python
from stride_s5 import run_s5, build_s5

# build + write artifacts
tables, report = run_s5(
    "outputs/stride_table.parquet",
    "outputs_s1a/conservation_table.parquet",
    "outputs_s5",
)

# build in memory only (no writes)
tables, report = build_s5(
    "outputs/stride_table.parquet",
    "outputs_s1a/conservation_table.parquet",
)
assert report.all_passed
tables.position_conservation      # the four tables are attributes of S5Tables
tables.direction_concordance
tables.domain_serotype_matrix
tables.cross_serotype_scorecard
```

Reusing individual builders in a later stage:

```python
from stride_s5.build import (
    build_position_conservation,
    build_direction_concordance,
    build_domain_serotype_matrix,
    build_cross_serotype_scorecard,
)
```

## Consumption map (quick reference)

| Table | Primary later consumers |
|---|---|
| `position_conservation` | cross-serotype conservation map; serotype-divergent shortlist |
| `direction_concordance` | direction-agreement view over shared signed positions |
| `domain_serotype_matrix` | domain × serotype ρ heatmap; conserved catalytic-machinery check |
| `cross_serotype_scorecard` | per-serotype summary row; shared-position reproducibility comparison |

S5 hands later stages a trustworthy, purely structural cross-serotype reduction.
No calibrated verdicts, no pseudoreplicated tests, and no figures happen here.
