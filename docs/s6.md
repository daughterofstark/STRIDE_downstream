# Stage S6 — the replicate layer

S6 is the **per-replicate-observation** layer. It is the only stage that consumes
the replicate-specific inputs: the S1A `replicate_inventory` (the
replicate-structure index — how many runs contain each `canon_label` per serotype,
and whether the replicate set is complete) and, when it exists, the S0
`replicate_table` (the Level-1 per-run observations — one row per
`(serotype, replicate, canon_label)` carrying that run's effect `r`, i.e. the
per-run θ). From these it produces the **replicate-axis** products of design §3.1:
a per-serotype **replicate-regime** ledger, a descriptive **per-run effect
spread**, a per-serotype **rank concordance** (Kendall's *W* / mean pairwise
Spearman), and a machine-readable **blocked-analysis ledger**.

> **What S6 is not.** S6 does **not** re-implement the **τ²-based**
> replicate-disagreement mapping or the τ²/σ̄² ratio ("replicate-dominated vs
> sampling-dominated") diagnostic of design §3.1 — those read the *aggregate*
> variance components from the STRIDE table and are already produced by **S4**
> (`residue_variance` ranks positions by τ²; `variance_budget` carries the
> τ²/σ̄² regime per domain). S6 never reads `stride_table`; it owns the
> *per-replicate* axis, not the aggregate one. It makes **no calibrated pass/fail
> claim** (the gate is uncalibrated, `calibrated = false`), reads per-run θ but
> never recomputes it, does no machine learning, and produces no figures.

S6 never consumes the S1B/S2/S3/S4/S5 outputs, never re-reads the raw STRIDE
`profile.csv` / `mechanism.json` or MD trajectories, and never modifies an earlier
stage.

---

## Input

| Input | Produced by | Required? | Columns S6 reads |
|---|---|---|---|
| `replicate_inventory.parquet` | S1A | **yes** | `serotype, canon_label, n_replicates, in_all_replicates` |
| `replicate_table.parquet` | S0 | optional | `serotype, replicate_index, canon_label, r` (+ `domain_label` for context) |

The inventory is always present: S1A emits it even when no per-run correlation
CSVs were supplied to S0 (every `n_replicates == 0`, `available == False`). The
replicate table is **optional** — S0 only writes it when the Level-1
`*_correlations_v5.csv` files were supplied. Its absence is the design's
anticipated "per-run θ unavailable" state (§4.1), **not** an error: the per-run
analyses are then recorded as blocked rather than approximated.

---

## What is supported, and what is blocked

Design §3.1 ("Replicate-level analysis") lists four analyses. S6's scope is
determined by which inputs actually exist:

| Design §3.1 analysis | Where it lives | Status in S6 |
|---|---|---|
| Replicate-disagreement mapping via τ² (rank by τ²) | **S4** `residue_variance` (`tau2_rank`) | delegated to S4 — not re-done here |
| τ²/σ̄² ratio diagnostic (replicate- vs sampling-dominated / metastability) | **S4** `variance_budget` / `residue_variance` (`variance_regime`) | delegated to S4 — not re-done here |
| ☞ Per-run θ rank concordance (per-run correlation / Kendall's *W*) | **S6** `replicate_concordance`, `replicate_effect_spread` | **available iff** the replicate table carries per-run θ; **blocked** otherwise |
| ☞ Leave-one-replicate-out stability of ρ / gated scale | — | **always blocked** |

The τ²-based items (1–2) read the K-aggregate and were already shipped in S4, so
S6 does not duplicate them; `docs/architecture.md` and this page point back to S4.
The per-run items (3–4) are what the design flags with ☞ as needing the per-run
correlation CSVs (§4.1) — they are S6's subject.

### Blocked: per-run concordance / spread when the CSVs are absent

When the per-run correlation CSVs were not supplied, S0 writes no
`replicate_table`, so no per-run θ exists in the pipeline. Per design §4.2, per-run
θ is **not reconstructible** from the K-aggregate τ²/σ̄². S6 therefore emits an
**empty** `replicate_concordance` and `replicate_effect_spread`, sets
`per_replicate_effects_available = false`, and records both analyses as `blocked`
in the ledger with the required input named. It never fabricates per-run values.

### Blocked: leave-one-replicate-out stability

Leave-one-replicate-out stability of ρ and of the gated scale is **always
blocked**. Recomputing ρ and re-gating on replicate subsets requires re-running
STRIDE on those subsets; only the K-aggregate is in the pipeline (design §3.1 ☞,
§4.2). This read-only pipeline never re-runs STRIDE, so the analysis is recorded as
`blocked` unconditionally and is never approximated (e.g. no per-run-θ-mean
surrogate is substituted for the real re-gated ρ).

## Core concepts

### Per-run effect availability

A serotype has **usable per-run effects** when the replicate table exposes a
finite θ in at least `MIN_REPLICATES_FOR_CONCORDANCE` (2) runs. The global
`per_replicate_effects_available` flag is true when *any* serotype meets this bar.
It gates the concordance and spread tables and the status of the two per-run
analyses in the ledger.

### Complete-case positions

Rank concordance across runs is computed over the positions observed with a finite
θ in **every** run of a serotype (the complete-case intersection). A serotype with
fewer than `MIN_POSITIONS_FOR_CONCORDANCE` (3) complete positions is reported with
an `insufficient` class and `nan` coefficients rather than a fabricated number.

### Residue-scale licensing (K = 3)

STRIDE's operating-range guidance licenses residue-scale claims only at K ≥ 5. At
the design's K = 3 the per-run spread and concordance are **residue-scale and
exploratory** (`tier = exploratory`), and `residue_claims_licensed` is `false`.

## Outputs

All tables are deterministic (sorted keys; floats rounded to 6 decimals).

### `replicate_regime.parquet` — one row per serotype

The replicate-structure ledger: `n_replicates` (K), `n_positions`,
`n_positions_in_all_replicates`, `frac_complete`, `residue_claims_licensed`
(K ≥ 5), `per_replicate_effects_available`, and `n_replicates_with_effects`.

### `replicate_effect_spread.parquet` — one row per observed `(serotype, canon_label)` (Tier B)

A *descriptive* across-run summary of each position's per-run θ: `n_obs`,
`theta_mean`, `theta_sd`, `theta_min`, `theta_max`, `theta_range`,
`abs_theta_mean`, `max_pairwise_abs_diff`, `in_all_replicates`, `tier`. Empty when
no per-run θ is available. This is descriptive only — it makes no inferential
claim.

### `replicate_concordance.parquet` — one row per serotype with effects (Tier B)

The per-run rank concordance (design §3.1): `n_replicates_with_effects`,
`n_positions_complete`, `kendalls_w` (Kendall's coefficient of concordance in
`[0, 1]`, or `nan`), `mean_pairwise_spearman` (in `[-1, 1]`, or `nan`),
`concordance_class ∈ {strong ≥ 0.7, moderate ≥ 0.4, weak, insufficient}`, `tier`.
Empty when no per-run θ is available.

### `replicate_blocked_analyses.parquet` — one row per replicate-level analysis

The explicit ledger so the blocked subset is documented rather than approximated:
`analysis_id`, `description`, `status ∈ {available, blocked}`, `available`,
`reason`, `required_input`, `design_ref`. The `per_replicate_rank_concordance` and
`per_replicate_effect_spread` rows track availability; the
`leave_one_replicate_out_stability` row is always `blocked`.

### `replicate_summary.json`

The provenance header (`calibrated = false`, the K = 3 note, the per-run-effect
availability, and the input digests), the four table sizes, compact facts
(K per serotype, licensed / with-effects counts, concordance-class counts, blocked
/ available analysis counts), the echoed blocked-analysis ledger, and the
structural validation outcomes.

## Kendall's *W* (deterministic, no SciPy)

For *m* runs ranking *N* complete-case positions, with average-rank tie handling
and the standard tie correction,

```
W = 12·S / ( m²·(N³ − N) − m·Σ T_j )
```

where `S = Σ_i (R_i − mean R)²`, `R_i` is the sum over runs of position *i*'s
average rank, and `T_j = Σ (t³ − t)` sums over the tie groups of run *j*. *W* is
`nan` for `N < 2` or `m < 2`, and `1.0` when the denominator collapses because
every run is internally constant. `mean_pairwise_spearman` is the mean Spearman
correlation over the `C(m, 2)` run pairs. All computations are pure Python over
plain lists — deterministic, no randomness, no SciPy dependency.

---

## Validation (structural only)

`stride_s6.validation` performs structural / arithmetic checks only — no
statistical assertions and no biological claims: unique keys on every table;
regime completeness bounded and `frac_complete` / `residue_claims_licensed` /
`per_replicate_effects_available` consistent with the counts; spread rows ordered
(`theta_min ≤ theta_max`) and non-negative; concordance coefficients in range and
the class matching the recomputed label; and the ledger's `available` flag
matching its `status`, with a leave-one-replicate-out row that is always blocked.

## CLI

```bash
python -m stride_s6 \
  --input-dir outputs \
  --inventory-input-dir outputs_s1a \
  --output-dir outputs_s6
# or the installed console script:
stride-s6 --input-dir outputs --inventory-input-dir outputs_s1a --output-dir outputs_s6
```

The S1A replicate inventory is read from `--inventory-input-dir` and the optional
S0 replicate table from `--input-dir`; each can be overridden with an explicit
path (`--replicate-inventory`, `--replicate-table`). Pass `--no-replicate-table`
to force the documented blocked path even when the table exists. S6 exits non-zero
only on a genuine input or consistency error; the absence of per-run θ is a
supported, documented outcome, not a failure.
